## 📋 榜单说明

### 🔥 一周卷王榜
**计算规则**：基于最近7天的GitHub commit活动综合评分
- **基础分**：每个commit = 1分
- **连续性奖励**：连续7天+10分，5天+5分，3天+2分
- **多仓库奖励**：参与5个以上仓库+5分
- **质量奖励**：平均每日5个以上commit+8分，2个以上+2分
- **数据源**：GitHub API实时获取
- **上榜条件**：至少1个commit
- **更新频率**：每日自动更新
- **筛选支持**：支持按研究方向和显示数量筛选

### 🌙 夜猫榜
**计算规则**：基于最近7天深夜时段（22:00-06:00）的GitHub commit活动评分
- **基础分**：每个深夜commit = 2分（比普通commit更有价值）
- **深夜比例奖励**：深夜提交占比50%以上+10分，30%以上+5分，20%以上+2分
- **连续性奖励**：连续5天以上有深夜提交+8分，3天以上+4分
- **多仓库奖励**：深夜参与3个以上仓库+3分，2个以上+1分
- **时区处理**：自动转换UTC时间为北京时间进行统计
- **数据源**：GitHub API commit时间戳分析
- **上榜条件**：至少1个深夜commit
- **更新频率**：每日自动更新
- **筛选支持**：支持按研究方向和显示数量筛选

### 🏆 人气王榜
**计算规则**：综合考虑个人社交影响力和组织仓库影响力
- **权重分配**：Followers × 0.6 + 组织仓库总Stars × 0.4
- **数据来源**：
  - Followers：个人 GitHub 关注者数量
  - 组织仓库总Stars：贡献者参与的所有 Datawhale 组织仓库的 stars 总和
- **更新频率**：每日更新
- **上榜条件**：至少有组织贡献或拥有 followers

### 💼 多产榜
**计算规则**：基于参与 Datawhale 组织仓库数量排名
- **评分标准**：参与的组织仓库数量
- **数据来源**：贡献者在 Datawhale 组织中有代码贡献的仓库数量
- **更新频率**：每日更新
- **上榜条件**：至少参与 1 个组织仓库

### 🤝 社交达人榜
**计算规则**：基于 GitHub following 数量
- **评分标准**：Following 数量
- **数据来源**：个人 GitHub 账号关注的用户数量
- **更新频率**：每日更新
- **上榜条件**：至少关注 10 个用户

### ⭐ 新星榜
**计算规则**：综合组织贡献活跃度指标，适合新加入的贡献者
- **评分公式**：(Followers + 组织仓库总Stars) / max(组织仓库数, 1) × 新人加成系数
- **新人加成系数**：参与少于 5 个组织仓库时获得 1.5 倍加成
- **数据来源**：基于 Datawhale 组织仓库的贡献数据
- **更新频率**：每日更新
- **上榜条件**：有组织贡献且活跃度评分 > 0

### 🌟 综合实力榜
**计算规则**：基于组织贡献的多维度综合评分
- **评分维度**：
  - 组织技术影响力：组织仓库总Stars × 0.3
  - 个人社区影响力：Followers × 0.25
  - 组织参与广度：参与组织仓库数 × 0.2
  - 社交活跃度：Following × 0.15
  - 代码贡献量：组织总贡献数 × 0.1
- **数据来源**：主要基于 Datawhale 组织仓库的贡献数据
- **更新频率**：每日更新
